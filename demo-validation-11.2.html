🧪 Test Scenario 1: Task Manager

Try These Tests:

  1. Empty Task: Click "Add" without typing anything
    • Expected: Red error message appears below the form
    • Expected: "Task description cannot be empty."
    • Expected: Focus stays in the input field
  2. Whitespace Task: Type only spaces and click "Add"
    • Expected: Same error message appears
    • Expected: No task is added to the list
  3. Valid Task: Type "Buy groceries" and click "Add"
    • Expected: Task is added successfully
    • Expected: No error message
    • Expected: Input field is cleared

🧪 Test Scenario 2: Quick Links

Try These Tests:

  1. Empty Name: Leave name empty, enter URL, click "Add Link"
    • Expected: Error message "Link name cannot be empty."
    • Expected: Focus moves to name input field
  2. Empty URL: Enter name, leave URL empty, click "Add Link"
    • Expected: Error message "Link URL cannot be empty."
    • Expected: Focus moves to URL input field
  3. Invalid URL: Enter name "Google", URL "google.com" (no protocol)
    • Expected: Error message with example: "URL must start with http:// or https:// (e.g., https://example.com)"
    • Expected: Focus moves to URL input field
  4. Valid Link: Enter name "Google", URL "https://google.com"
    • Expected: Link is added successfully
    • Expected: No error message
    • Expected: Input fields are cleared

♿ Accessibility Features

  • ✓ ARIA Attributes: Error messages use role="alert" and aria-live="polite"
  • ✓ Screen Reader Support: Errors are announced to assistive technologies
  • ✓ Keyboard Navigation: Focus management ensures keyboard users stay on invalid fields

  • ✓ Visual Cues: Warning icon (⚠) provides non-color indicator
