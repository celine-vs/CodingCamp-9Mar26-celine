Task 11.2 - Input Validation and Feedback Testing

Task Requirements:
  • Add inline validation messages for empty inputs
  • Add validation message for invalid URL format with example
  • Prevent form submission when validation fails
  • Maintain focus on invalid input fields
  • Requirements: 3.8, 5.5, 5.6

Automated Tests

Manual Testing Instructions

Task Manager Validation Tests

  1. Empty Task Test: Try to add a task with an empty description
    • ✓ Should show error: "Task description cannot be empty."
    • ✓ Focus should remain on the task input field
    • ✓ No task should be added to the list
  2. Whitespace Task Test: Try to add a task with only spaces/tabs
    • ✓ Should show error: "Task description cannot be empty."
    • ✓ Focus should remain on the task input field
    • ✓ No task should be added to the list
  3. Valid Task Test: Add a task with valid text
    • ✓ Should add the task successfully
    • ✓ No error message should appear
    • ✓ Input field should be cleared

Quick Links Validation Tests

  1. Empty Name Test: Try to add a link with empty name
    • ✓ Should show error: "Link name cannot be empty."
    • ✓ Focus should move to the name input field
    • ✓ No link should be added
  2. Empty URL Test: Enter a name but leave URL empty
    • ✓ Should show error: "Link URL cannot be empty."
    • ✓ Focus should move to the URL input field
    • ✓ No link should be added
  3. Invalid URL Format Test: Enter "example.com" (no protocol)
    • ✓ Should show error: "URL must start with http:// or https:// (e.g., https://example.com)"
    • ✓ Focus should move to the URL input field
    • ✓ No link should be added
  4. Valid Link Test: Add a link with valid name and URL
    • ✓ Should add the link successfully
    • ✓ No error message should appear
    • ✓ Input fields should be cleared
